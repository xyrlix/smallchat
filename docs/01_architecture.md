# SmallChat 架构文档

## 概述

SmallChat是一个基于TCP的轻量级聊天服务器，采用多线程架构，支持多客户端并发连接和实时消息传递。

## 服务器架构

### 核心组件

#### 1. 主线程 (Main Thread)
- 创建服务器socket
- 监听客户端连接
- 启动accept线程
- 处理信号（SIGINT, SIGTERM）

#### 2. Accept线程
- 持续接受新的客户端连接
- 为每个客户端创建处理线程
- 发送欢迎消息

#### 3. 客户端处理线程 (Client Handler Threads)
- 每个客户端一个独立线程
- 处理客户端消息
- 解析和执行命令
- 转发消息

#### 4. 共享数据结构
- `clients_`: 客户端映射表（socket_fd -> ClientInfo）
- `name_to_socket_`: 名称映射表（name -> socket_fd）
- 使用互斥锁保护

### 数据流

```
Client A                    Server                      Client B
   │                          │                             │
   │ ───── MSG ────────────> │ ───── MSG ────────────────> │
   │                          │                             │
   │ <───── Broadcast ─────── │                             │
   │                          │                             │
   │ <───── OK/ERROR ───────── │                             │
   │                          │                             │
   │ ───── /whisper ────────> │ ───── MSG ────────────────> │
   │                          │                             │
   │                          │ <───── ACK ───────────────── │
   │                          │                             │
```

## 客户端架构

### 核心组件

#### 1. 主线程 (Main Thread)
- 读取用户输入
- 发送消息到服务器
- 处理命令

#### 2. 接收线程 (Receive Thread)
- 异步接收服务器消息
- 解析消息类型
- 调用回调函数

#### 3. 回调机制
- `message_callback_`: 处理普通消息
- `system_callback_`: 处理系统消息
- `userlist_callback_`: 处理用户列表
- `error_callback_`: 处理错误消息

### 数据流


## 协议设计

### 消息格式

所有消息以换行符 `\n` 结尾，字段以竖线 `|` 分隔。

### 消息类型

#### 1. 普通消息 (MSG)
```
MSG|sender|receiver|type|content\n
```

**字段说明**:
- `sender`: 发送者名称
- `receiver`: 接收者名称（空表示广播）
- `type`: 消息类型（0=BROADCAST, 1=PRIVATE, 2=TEXT, 3=SYSTEM）
- `content`: 消息内容

**示例**:
```
MSG|Alice||0|Hello everyone!
MSG|Alice|Bob|1|Hi Bob!
```

#### 2. 系统消息 (SYS)
```
SYS|message\n
```

**示例**:
```
SYS|Alice joined the chat
SYS|Server maintenance at 00:00
```

#### 3. 用户列表 (USERS)
```
USERS|count\n
user1\n
user2\n
...
```

**示例**:
```
USERS|3\n
Alice\n
Bob\n
Charlie\n
```

#### 4. 错误消息 (ERROR)
```
ERROR|error_message\n
```

**示例**:
```
ERROR|Name already taken
ERROR|User not found
```

#### 5. 成功消息 (OK)
```
OK|message\n
```

**示例**:
```
OK|Logged in as Alice
OK|Goodbye!
```

#### 6. 欢迎消息 (WELCOME)
```
WELCOME\n
welcome_message\n
commands\n
```

**示例**:
```
WELCOME\n
Welcome to SmallChat Server!\n
Commands: /login <name>, /users, /whisper <name> <msg>, /help, /quit\n
```

## 线程安全

### 互斥锁使用

1. **clients_mutex_**: 保护 `clients_` 和 `name_to_socket_`
   - 添加/删除客户端时加锁
   - 查询客户端时加锁
   - 广播消息时加锁

2. **threads_mutex_**: 保护 `client_threads_`
   - 添加线程时加锁
   - 清理线程时加锁

### 锁策略

- **细粒度锁**: 尽量缩小锁的范围
- **读写分离**: 查询和修改分离
- **避免死锁**: 始终按相同顺序获取锁

## 错误处理

### 服务器错误

1. **连接失败**
   - Socket创建失败
   - Bind失败
   - Listen失败

2. **客户端错误**
   - 连接断开
   - 消息发送失败
   - 名字冲突

3. **协议错误**
   - 消息格式错误
   - 字段缺失
   - 类型不匹配

### 客户端错误

1. **连接错误**
   - 无法连接到服务器
   - 连接超时
   - 连接断开

2. **协议错误**
   - 收到无效消息
   - 消息解析失败

## 性能优化

### 当前实现

1. **非阻塞I/O**: 使用非阻塞socket
2. **多线程**: 每个客户端独立线程
3. **连接池**: 复用连接
4. **零拷贝**: 直接发送消息

### 未来优化

1. **事件驱动**: 使用epoll/kqueue
2. **连接池管理**: 限制最大连接数
3. **消息队列**: 异步消息处理
4. **压缩**: 消息压缩传输
5. **缓存**: 用户信息缓存

## 总结

SmallChat采用了简洁的架构设计，通过多线程和互斥锁实现了并发安全，使用简单的文本协议保证了易用性和扩展性。该架构适合中小型聊天应用，可作为学习和扩展的基础。
